# Reports Directory

本目录按项目阶段存放可汇报、可复核的分析产物。

| 目录 | 阶段 | 内容 |
| --- | --- | --- |
| `P1/` | P1 系统对比 | Table 1/2、A5/A6 分析、可视化图、PPT 和 HTML 汇报材料 |
| `P2/` | P2 MLOps 实践 | 超参数优化、模型注册、部署评估、监控和验收报告 |

约定:

- 阶段内产物统一放在对应子目录,避免 `reports/` 根目录混放。
- 实验脚本应写入阶段目录,例如 P1 写入 `reports/P1/`。
- P2 不只依赖 MLflow/Optuna DB,关键结论也应导出为表格或 Markdown,便于汇报和复核。
