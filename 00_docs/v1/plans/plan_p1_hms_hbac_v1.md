## 摘要

本项目拟基于 Kaggle `HMS - Harmful Brain Activity Classification` 竞赛数据，面向 ICU 场景下 EEG 有害脑活动识别任务，设计一个适合课程项目 1 的多系统对比方案。由于原始 HMS 任务更偏深度学习，不适合直接在 `sklearn / Spark / PyTorch` 上进行公平横向比较，本计划将采用“频谱统计特征化”的策略，将原始 EEG 或 spectrogram 数据转换为结构化特征，再统一在单机、分布式和深度学习系统中实现 `Logistic Regression` 与 `MLP`。项目重点在于回答: 对于医疗信号任务，当输入被转为结构化频谱特征后，不同系统在性能、资源占用和工程复杂度上呈现何种差异。项目同时将保留一版轻量深度频谱模型作为附加展示，以增强汇报表现力。

关键词: `EEG`、`HMS`、`频谱特征`、`多系统对比`、`医疗信号`

## 1. 研究背景与意义

EEG 有害脑活动识别是典型的高价值医疗信号任务。公开资料显示，该 Kaggle 任务涉及约 `1950` 名患者、`106,800` 个 EEG 片段，并需在 `6` 类脑活动模式之间进行判别。与传统课程任务相比，该题目具有更强的研究味和临床应用价值。

但从课程项目 1 的角度看，直接做端到端频谱深度学习会削弱系统对比的公平性。因此本项目拟通过结构化特征化处理，将高复杂度医疗信号任务转化为标准机器学习系统实验对象。

## 2. 任务理解与问题重构

原竞赛本质上更适合 `CNN` 或频谱深度网络，而课程项目 1 要求在三类系统中实现同类算法。因此本项目采用如下重构:

- 从 EEG 频谱或 spectrogram 中提取统计、纹理和频带能量特征。
- 将样本转化为标准表格输入。
- 在 `sklearn / Spark / PyTorch` 中分别实现 `LR` 与 `MLP`。

这样既保留 HMS 数据的专业性，又满足课程项目的系统对比要求。

## 3. 研究目标

1. 完成 HMS 数据的结构化特征提取。
2. 在三类系统中实现统一算法实验。
3. 评估医疗信号任务中结构化建模的可行性。
4. 分析深度系统和传统系统在该任务上的边际差异。

## 4. 研究内容

- 读取竞赛标签文件与频谱数据
- 构建频带能量、统计矩和纹理等结构化特征
- 完成标签映射与类别平衡分析
- 实现三类系统的统一对比
- 可选地增加一版轻量 CNN 作为展示模型

## 5. 技术路线

1. 数据整理与样本对齐。
2. 频谱特征提取。
3. 结构化训练集构建。
4. `LR` 和 `MLP` 三系统实现。
5. 输出精度、耗时和资源对比。

### 7.2 方案 B: 深度展示增强

- 风险 1:

  - 应对: 引入加权损失和分层划分
